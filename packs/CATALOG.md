## What Are Pharos Packs?

Walk-encoded knowledge graphs injected into a model's KV cache at inference time. No fine-tuning, no retraining, no API calls. The model gains domain expertise in milliseconds, and the packs work across any open-weight architecture.

Think "I know kung fu" — but for mutual aid workers, legal advocates, crisis counselors, and researchers.

## Verification

Every non-philosophy pack includes:
- `verification.date` — when last checked
- `verification.confidence` — 0-1 accuracy rating
- `source_urls` — 5-15 authoritative sources per pack
- `pending_corrections` — flagged issues awaiting review

Quarterly automated verification via `verify_critical_packs.py` checks:
- Critical service URLs (crisis hotlines, resource pages)
- Overdue verification dates by schedule tier
- Alerts via NATS to Coalition Discord

## How Packs Work

```
1. Agent receives user query
2. Gateway selects relevant packs by domain
3. Pack triples are walk-encoded into the KV cache
4. Model generates with domain expertise — zero additional tokens
5. No fine-tuning, no API calls, model-agnostic
```

Each pack is a JSON file with `{description, source, triples, verification, source_urls}`.
Triples follow the `{subject, predicate, object}` schema.
Every pack includes `agent protocol` rules for safe deployment.
